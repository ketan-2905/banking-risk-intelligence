4Transfer Scenarios
Run in order: LOW → MEDIUM → HIGH.
LOW → small transfer to known beneficiary. ACCOUNT_AGE_UNDER_7_DAYS (+20) → score 20 → LOW → POSTED immediately.
MEDIUM → new beneficiary + login anomaly. NEW_BENEFICIARY (+25) + ACCOUNT_AGE_UNDER_7_DAYS (+20) + LOGIN_ANOMALY (+16) → score 61 → MEDIUM → POSTED.
HIGH → run MEDIUM first to establish $40 hourly velocity, then HIGH adds $5,100 → total $5,140 > $5,000 threshold. All MEDIUM signals + HOURLY_VELOCITY_OVER_5000 (+30) → score 91 → HIGH → escrow → PENDING_REVIEW.

5Analyst Review
High-risk transfers do not move funds immediately. They wait for analyst review. Requires analyst token.
Approve → held funds settle to destination account.
Reject → held funds are released back to source account.
Click an alert card to select it, then approve or reject.
Click "List pending alerts" after running the HIGH-risk transfer in Step 4. Each alert card has inline Approve / Reject buttons. If the list is empty, click "Run HIGH transfer + list alerts" above (requires Step 2 context + customer2 token).
6Observability
Health is public. Metrics and Prometheus are protected and require an analyst JWT. Get analyst token in Step 3 first.
Opening /actuator/metrics or /actuator/prometheus directly in the browser without a token returns {"code":"UNAUTHORIZED"} — this is expected security behavior, not a bug.
